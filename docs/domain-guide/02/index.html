02. PreDist 데이터셋 가이드

왜 이 문서를 읽어야 하는가

논문을 이해해도 실제 데이터 구조를 모르면 모델과 Agent를 설계할 수 없다.

데이터셋을 볼 때 확인할 것

  • 파일 구성이 어떻게 나뉘는지
  • 기계실별 데이터가 어떻게 구분되는지
  • 시계열 간격이 어떤지
  • 라벨이 어느 단위로 주어지는지
  • 유지보수/고객신고 정보와 센서 데이터가 어떻게 연결되는지

핵심 해설

  • PreDist는 기계실별로 운영 데이터가 쌓이는 구조이므로 다중 엔티티 시계열 문제로 보는 게 맞다.
  • 따라서 HeatGrid는 단일 설비 모델보다 기계실 ID를 가진 멀티테넌트 구조로 설계해야 한다.

HeatGrid에 직접 쓰는 포인트

  • 입력: 기계실별 센서 시계열
  • 중간산출: anomaly score, 리드타임, 영향도
  • 출력: 우선순위, 작업지시, 재계획

PreDist/Agent 설계와 연결

데이터 요소설계 연결
기계실 ID멀티 엔티티 운영
시계열 센서값이상탐지/리드타임 모델
maintenance 라벨작업 결과 로그
customer report민원/체감 이벤트

초심자 체크포인트

  • 이 데이터가 왜 기계실 하나가 아니라 여러 기계실 문제인지 설명할 수 있는가
  • 어떤 라벨이 Agent 설계에 중요한지 말할 수 있는가
